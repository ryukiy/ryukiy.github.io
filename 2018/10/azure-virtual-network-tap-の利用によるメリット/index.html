Azure Virtual Network TAP の利用によるメリット

2018-10-25 カテゴリー: Microsoft Azure, Virtual Network

Microsoft Ignite 2018 において、Azure Virtual Network TAP (VTAP) のプレビューが発表された。VTAP を使用した場合の効果について触れてみる。

TAP とは何か？

TAP とは Terminal Access Point の略であり、ネットワーク機器がトラフィックを受け取るためのアクセス ポイントを意味する。TAP を利用した場合、ネットワークのトラフィックがミラーリングされ、既存のネットワーク構成を変更することなくトラフィックの監視が可能となる。ここでは、既存のネットワーク環境に対し、次世代ファイアウォールを新たに設置してネットワーク経由のサイバー攻撃の状況を監視する例を挙げる。 ネットワーク機器の構成変更はリスクが伴うため、リスクを回避する手段として TAP が用いられている。このように、TAP は Azure 特有のものではなく、従来からあるネットワークのテクニックとして利用されている。

現状の Azure VNET におけるチャレンジ

VNET に対するサイバー攻撃の監視を行う目的でセキュリティ解析を行う NVA (Network Virtual Appliance) を設置する場合、Azure VM でホストされるワークロードのトラフィックが NVA を経由（インライン構成）するようにデザインする必要がある。この構成については、Azure のドキュメント で紹介されているので、詳細はこちらのページを参考にしていただきたい。

留意が必要になるケースとして、既存の VNET に新規で NVA を設置するパターンが挙げられる。ここでは、フロントエンドに Web、バックエンドに DB が存在する環境の例で考えてみる。既存の VNET に NVA を設置するためには、以下のような構成変更が必要となる。

この構成変更のポイントは以下の通りとなる。

  • VNET に NVA を設置するためのサブネット（この例では Security サブネット）を追加し、DMZ の役割を割り当てる
  • NVA には 3 つの NIC を追加し、それぞれを Security/Web/DB サブネットに所属させる
    • この構成により、NVA はサブネット間のトラフィックを受け取ることが可能となる
  • Web/DB サブネットに UDR を設定し、Next Hop に NVA の NIC を指定する
    • UDR を設定し、VNET から提供されるルーティング情報を上書きすることで、サブネット間のトラフィックを NVA に送信することが可能となる

この構成変更におけるチャレンジとして、UDR によるルーティングの変更による影響が挙げられる。UDR はサブネット内に存在する全ての VM のルーティングに影響するため、本番環境に UDR を設定する際には、Next Hop が NVA になった場合のサイド エフェクトを十分に考慮する必要がある。

UDR の代替手段として、Azure VM のゲスト OS 内のルート テーブルを変更するという方法がある。この方法の場合、NVA でトラフィックを監視したいワークロードに関連する VM のみが変更対象となるため、サブネット内のその他の VM のルーティングは影響を受けないというメリットがある。ただし、監視対象のワークロードをホストする VM の台数が多いような場合には、ゲスト OS のルート テーブルを書き換える作業に工数がかかる点を考慮しなくてはならない。

VTAP によるメリット

先で述べたように、TAP を利用することによるメリットは、トラフィックをミラーリングすることにより既存のネットワーク構成を変更せずに、ネットワーク経由のサイバー攻撃を監視できることにある。これを Azure VNET で実現するのが VTAP である。

VTAP を利用することで、UDR によるルーティングの変更を行う必要がないため、VNET 上に存在する既存の VM のルーティングに影響を与えることなく NVA でトラフィックの監視を行うことが可能となる。

VTAP を利用する際の注意事項として、サイバー攻撃からの防御が利用目的ではない点に留意する必要がある。VTAP によって提供されるトラフィックはミラーリングされたものであるため、セキュリティ解析を行う NVA が VTAP 経由で攻撃を検知したとしても、攻撃のパケットはすでにワークロードを提供する VM に到達している。そのため、VTAP を利用する目的としては、「攻撃からの防御」ではなく、「攻撃状況の監視」であることに留意していただきたい。

また、VTAP はトラフィックをミラーリングする際の方式として VXLAN を採用している。そのため、NVA の設置を検討する際には、NVA が VXLAN をサポートしているかどうかについて、ご確認いただきたい。

現在のステータス

この記事を執筆している段階（2018 年 10 月 25 日）では、米国中西部リージョンでプレビューが実施されている。VTAP を利用する際には、サブスクリプション ID を指定して事前にプレビューに登録する必要がある。NVA の設置方式にお困りの方は、プレビューに登録して VTAP を試していただきたい。
